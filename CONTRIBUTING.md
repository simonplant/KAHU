# Contributing to KAHU

Thank you for considering contributing to KAHU — a commons-based AI guardian for human consciousness development.

This project is in its founding phase. Every contribution shapes what KAHU becomes.

---

## Ways to Contribute

### Vision and Philosophy
- Refine the core vision and manifesto documents in `vision/`
- Propose additions from your wisdom tradition or cultural background
- Identify gaps or contradictions in the philosophy
- File an issue or open a discussion

### Technical Development
- Build toward the guardian prototype (see `prototypes/guardian-mvp/`)
- Improve the technical architecture (`architecture/`)
- Write or review code for memory, emotional intelligence, and proactive care systems
- Set up local Llama-based inference and document your experience

### Community and Governance
- Help design democratic governance processes (`governance/`)
- Draft community guidelines and codes of conduct
- Facilitate discussions and help resolve disagreements
- Connect consciousness development communities with the project

### Cultural Advisory
- If you carry knowledge from Hawaiian or other wisdom traditions, we especially welcome your guidance
- Help ensure KAHU honors the *kahu* concept respectfully and not appropriatively
- Review documents for cultural sensitivity issues and file issues

### Research
- Academic work on consciousness development, AI ethics, or guardian-type relationships
- Literature reviews of relevant wisdom traditions
- Studies validating (or questioning) guardian AI as a concept

---

## Getting Started

1. **Read** the [vision documents](./vision/) and [technical architecture](./architecture/)
2. **Browse** open [issues](../../issues) and [discussions](../../discussions)
3. **Pick something** that matches your skills and interests
4. **Talk to us** before doing significant work — open an issue or discussion first

## Principles for Contributing

### Consciousness First
Contributions should serve the development of human consciousness. When in doubt, ask: *does this serve users' flourishing, or something else?*

### Cultural Respect
KAHU draws deeply from Hawaiian cultural tradition. All contributions must handle cultural knowledge with care. If you're uncertain, ask before assuming.

### Transparency
Document your reasoning. KAHU is built on radical transparency. Share not just what you're doing but why.

### Democratic Process
Major decisions go through community discussion. Individual contributors don't unilaterally define direction — we figure it out together.

---

## Code Contributions

### Style
- Python: follow PEP 8, type hints preferred
- Documentation: clear, plain English
- Comments: explain *why*, not just *what*

### Process
1. Fork the repository
2. Create a branch: `git checkout -b your-feature-name`
3. Make your changes with clear commit messages
4. Open a pull request with a description of what you changed and why
5. Expect discussion — this is a community project

### Commit Messages
Use clear, present-tense messages:
- `Add guardian memory system prototype`
- `Fix emotional pattern recognition edge case`
- `Update architecture to reflect governance feedback`

## Questions and Discussion

- **GitHub Issues**: bugs, specific feature requests, concrete problems
- **GitHub Discussions**: broader questions, ideas, philosophical exploration
- **Email**: for sensitive matters or cultural advisory inquiries

---

## Code of Conduct

KAHU is built on the principle that every conscious being deserves care and respect. We expect that same standard in our community:

- Treat every contributor as a precious conscious being
- Disagree with ideas, never attack people
- Welcome newcomers with genuine warmth
- Acknowledge when you don't know something
- Give credit freely

We will remove contributors who violate these principles.
